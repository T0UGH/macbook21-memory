## 一、目标

先定一个小目标：

**每周稳定写 1 篇微信公众号文章。**

这件事的目标不是短期爆文，而是逐步建立一个清晰的个人标签：

> **一个真实经历 AI coding 变革、并且已经在团队中推广 Claude Code 的服务端工程师。**

这个视角是有稀缺性的。

---

## 二、为什么这个方向适合我

我的背景：

- 本职工作是互联网公司的服务端工程师
- 和 MT 的交流大多发生在业余时间
- 在团队里较早系统使用并推广 Claude Code
- 我会主动给团队成员讲 Claude Code，属于推动者/宣传者
- 我已经明显感受到：**本职工作里的 coding 部分，正在大幅被 Claude Code 替代**

所以，最适合我的内容方向，不是泛泛谈 AI 新闻，也不是写工具教程大全，而是：

> **从一线服务端工程师视角，记录 AI coding 正在如何改变工作流、团队协作和职业路径。**

这个方向的优势：

1. **有一手真实观察**，不是二手评论
2. **可持续写成系列**，不是一次性话题
3. **容易形成个人标签**，比泛 AI 内容更有辨识度
4. **能兼顾技术人群和泛职场人群**，有传播空间

---

## 三、公众号内容主线

建议围绕一个长期主线：

# AI 编程时代，一线工程师的真实工作流

核心只讲三件事：

1. **Claude Code 真实改变了什么**
2. **服务端工程师应该如何升级自己的位置**
3. **团队用了 AI 之后，真正的瓶颈和风险在哪里**

注意：

- 不要一上来就写大而空的 AI 评论
- 不要写成纯工具教程
- 不要堆 prompt 清单
- 要突出“真实经验 + 工程判断 + 职业升级”

## 六、第一篇文章结构建议

建议写成 1500~2500 字，结构不要太散。

### 1. 开头：从真实变化切入

建议开头直接讲清楚几件事：

- 我本职是服务端工程师
- 我不是 AI 抵触者，而是团队里较早推广 Claude Code 的人
- 但正因为我最早用，所以我最先感受到一个事实：
- **我工作里大量 coding，已经开始被 Claude Code 替代**

这里的重点不是煽情，而是建立真实可信的场景。

---

### 2. 第二部分：真正被替代的是什么

这里不要写成“程序员马上失业”，而要写得更准确：

被替代的不是“工程师这个职业”本身，而是：

- 标准化 coding 产能
- 低判断密度任务
- 明确输入输出的执行型工作
- 依赖手工堆时间的编码劳动

可以强调：

> AI 最先吃掉的，不一定是能力最弱的人，而是工作最标准、最可描述、最容易拆解的人。

---

### 3. 第三部分：我为什么开始重新定义自己的位置

这部分是全文最关键的转折。

要讲清楚：

如果 coding 正在被工具吞掉，那工程师的价值必须转移。

未来更重要的，不再只是“写得快”，而是：

- **方案能力**：问题怎么定义，系统怎么设计
- **验收能力**：AI 写出来的东西到底能不能进生产
- **流程能力**：怎么让 AI 真正提升团队产能，而不是制造更多技术债

这部分最好写得坚决一点，不要写成模棱两可的平衡态度。

---

### 4. 第四部分：这对普通工程师意味着什么

可以用更通俗的方式往外推一层：

- 如果你今天的主要价值是写标准代码，风险会越来越大
- 如果你能定义问题、做架构判断、识别风险、组织协作，你的价值会升高
- 以后真正稀缺的，不是“会不会写”，而是“知不知道该怎么做、敢不敢为结果负责”

---

### 5. 结尾：立住自己的系列方向

结尾建议不要喊口号，而是自然落在一个新的观察框架上。

比如：

- 我不会停止用 Claude Code，反而会继续更深地用
- 但我也不会再把“写代码”当成工程师最核心的价值
- 接下来，我会继续记录 AI coding 时代里，工程师应该如何升级自己

这样很适合引出后续连续写作。

## 七、给知更的写作要求

这部分很重要。

### 希望知更保留的东西

1. **真实一线工程师视角**
   - 不要写成泛 AI 评论
   - 不要写成媒体腔

2. **有工程判断，不要空泛鸡汤**
   - 重点是对工作流、职业价值、团队流程的真实判断

3. **有一点个人叙事，但不要过度煽情**
   - 重点不是“我焦虑了”，而是“我看到了变化，并做了判断”

4. **保持公众号可读性**
   - 标题、开头、转场、段落节奏可以更像成熟公众号文章
   - 但不要牺牲技术判断

5. **语气最好是真诚、克制、坚定**
   - 不要极端标题党
   - 不要夸张预言
   - 不要把文章写成情绪宣泄

## 八、不建议写成什么样

### 不要写成：

- AI 新闻综述
- Claude Code 使用说明书
- prompt 技巧大全
- “程序员还有没有未来”式空泛焦虑文
- 纯喊口号的职业鸡汤

### 要写成：

- 有现场感
- 有真实观察
- 有结构化判断
- 有明确结论
- 有后续连载空间
